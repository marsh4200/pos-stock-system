# POS Stock System

Self-hosted stock & barcode tracking for workshops, retail, and small businesses.
Built on Node.js + SQLite + React. Designed to run on a small Ubuntu VPS.

| Device         | Good?     | Notes                             |
| -------------- | --------- | --------------------------------- |
| Raspberry Pi 4 | Yes       | Fine for small shop/workshop      |
| Raspberry Pi 5 | Excellent | Fast enough for many users        |
| Intel NUC      | Excellent | More professional long-term       |
| Ubuntu VPS     | Excellent | Remote/cloud hosted               |
| Server         | Excellent | Best for multi-user / heavy loads |
| Old PC         | Yes       | Cheap self-host option            |


---
## ✨ Features

- USB barcode scanner workflow for issuing stock
- Per-operator badge codes (no per-staff login needed)
- Products, categories, suppliers, locations, min stock + reorder qty
- Live low-stock alerts
- Per-period archiving — close out each month with one click (or auto on the 1st)
- Per-operator, per-product reports for the current and any past month
- Backup & restore from the Settings page
- Customizable branding (name, subtitle, logo)
- Admin authentication with 5-minute inactivity timeout
- **One-click in-app updates from GitHub**

## 🚀 Install (one line)

On a fresh Ubuntu 22.04 / 24.04 server:

```bash
curl -sL https://raw.githubusercontent.com/marsh4200/pos-stock-system/main/install.sh | sudo bash
```

That's the whole install. Takes ~2 minutes.

When it finishes, open `http://your-server-ip:3001` and create your first admin account.

## 🔄 Updating

Once installed, you never need to SSH to update.

1. Log into the app
2. Go to **Settings → Updates**
3. Click **Check for Updates**
4. Click **Update Now**

Your database is automatically backed up before each update. If something goes
wrong, click **Roll back to previous version** on the same screen.

## 🛠️ Stack

- **Frontend**: React 18 + Vite + Tailwind + lucide-react
- **Backend**: Node.js 20 + Express + better-sqlite3 + bcryptjs + multer
- **DB**: SQLite with WAL mode, FK enforcement
- **Auth**: bcrypt + DB-backed Bearer tokens with sliding 5-min expiry
- **Source of truth**: this GitHub repo

## 📁 Data layout

Everything writable lives under `/opt/everton-stock/data/`:

- `everton.db` — main SQLite database
- `backups/` — auto-snapshots every 6 hours (last 30 kept) + pre-update snapshots
- `uploads/logo` — uploaded brand logo (if set)
- `updater.log` — live log of the last update or rollback

The install at `/opt/everton-stock/` is regenerated by every update.
The repo clone at `/opt/pos-stock-system/` is what the updater pulls from.
The database is never touched by updates.

## 🔌 Useful commands

```bash
sudo systemctl status everton-stock         # service status
sudo systemctl restart everton-stock        # restart after manual config changes
sudo journalctl -u everton-stock -f         # live logs
cat /opt/everton-stock/data/updater.log     # last update log
```

## 🌐 Putting it behind HTTPS

The installer runs the app on port 3001 in HTTP. To put it on a real domain
with HTTPS, use a Cloudflare Tunnel (free) or nginx + certbot. The app itself
doesn't need anything special — it just listens on 3001.
